              Exportar Histórico Completo

              Gera um arquivo .json com todas as versões de cargos registradas, assinado com hash manifesto SHA-256. Qualquer modificação externa ao arquivo é detectável matematicamente.

              O JSON inclui hashes encadeados e hash manifesto SHA-256. O PDF abre uma prévia formatada — use "Salvar como PDF" no diálogo de impressão do navegador.

              Garantia de Integridade

              O JoyDescription usa um sistema de hash encadeado inspirado na tecnologia blockchain para garantir que nenhuma versão de cargo possa ser alterada sem detecção.

              Como funciona o encadeamento

              Cada versão salva gera um hash SHA-256 calculado a partir do conteúdo do texto e do hash da versão anterior do mesmo cargo:

              hash = SHA256( hash_anterior + texto )

              Isso cria uma corrente: alterar qualquer versão do passado invalida todos os hashes subsequentes daquele cargo.

              Hash Manifesto do Export

              Ao exportar, o sistema gera um hash manifesto — SHA-256 de todos os hashes individuais concatenados em ordem:

              manifesto = SHA256( hash[1] | hash[2] | … | hash[n] )

              Esse valor fica registrado no arquivo. Se qualquer registro for removido ou reordenado, o manifesto não bate mais.

              Como verificar manualmente

              1. Abra o .json exportado em qualquer editor de texto
              2. Para cada versão: calcule SHA256(hash_prev + texto) e compare com o campo hash
              3. Para o manifesto: concatene todos os hash com | em ordem de id e calcule o SHA-256
              4. Qualquer divergência indica adulteração do arquivo

              O que esse sistema garante

              • ✓ Nenhuma versão de cargo pode ser silenciosamente editada após salva
              • ✓ A sequência de mudanças é imutável e auditável
              • ✓ O arquivo exportado é auto-verificável sem depender do servidor
              • ✓ Qualquer ferramenta que compute SHA-256 pode validar a cadeia

              O sistema não usa assinatura criptográfica com chave privada — a segurança protege contra adulteração acidental ou informal, não contra um atacante com acesso ao banco de dados.

              Backup manual

              Gera e baixa uma cópia completa do banco de dados agora. O arquivo pode ser guardado localmente, em nuvem ou enviado por e-mail.

              Restaurar backup

              Substitui o banco atual por um arquivo de backup. Todos os dados inseridos após o backup serão perdidos. O sistema reinicia automaticamente após a restauração.

              Backup automático

              Realiza checkpoint automático do banco em intervalos regulares e registra no histórico.

              Boas práticas
              • Faça backup antes de grandes alterações
              • Guarde ao menos os 3 últimos backups
              • Salve em local diferente do servidor (Drive, pendrive)
              • Rotina recomendada: semanal ou após cada sessão importante
              • Teste a restauração periodicamente em ambiente separado
